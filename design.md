# 语言设计

## 核心哲学

1. 操纵数据对象的模型

本语言的核心概念是阅读文档、操纵数据对象。

任何数据对象都需要给出相关的文档。

2. 使用词典对语言进行扩展

指令的集合、数据、数据的地址（后两者相当于特殊的指令），都可以被给予名字。

3. 编译为字节码，使用解释器进行执行

## 执行方式

使用一个虚拟机执行程序。虚拟机就是一块内存区域。

## 内置对象

输入流：键盘

输出流：终端行

键盘对象需要实现相当于 getwchar 的功能。

终端行对象需要实现相当于 putwchar 的功能。

这两个对象需要进行进一步的扩展，以实现更加底层的操作。

## 内置对象文档

键盘：

```
键盘 =
有输入 = #1/1B  /* 默认值为 0 。当有新的字符从键盘输入到终端时，值为 1 ，否则值为 0 。 */
输入   = #2/8B  /* 默认值为 0 。当有新的字符从键盘输入到终端时，值为该字符，否则值为默认值或者上一次输入的字符。 */
= 键盘
```

终端行：

```
终端行 =
    行   = #1~80/8B   /* 默认值为全 0 。当输出从 0 变为 1 时，其内容会被打印到终端 */
    输出 = #81/1B     /* 默认值为 0 。当从 0 变为 1 时，将行中的字符打印到终端，然后清空行中的内容 */
    格式 = #82~162/1B /* 每行的格式化掩码 格式-#1 对应于 行-#1 处字符的格式 */
    光标 = 1          /* 一般用法是 存 <字符> 到 终端行/行#终端行/光标 存 <格式> 到 终端行/格式#终端行/光标 */

    打印行 =
    存 0 到 终端行/输出
    存 1 到 终端行/输出
    = 打印行
= 终端行
```

```
等待输入 =
重复执行
检测条件 键盘/有输入
直到 成立
存 键盘/输入 到 终端行/行#终端行/光标
令 终端行/光标 加一
= 等待输入
```

### 5

地址访问的规则：任何对象的名称本质上都是一个地址。前面加上“#”的整数或者整型变量也是地址。

如果要保证地址访问的正确性，就要保证地址访问中的每个项都是一个地址。

现在出现这样一个问题：下面的 B 是对象 A 中的对象。要访问 B 的值，就要用 A 的地址作为基地址，B 的地址作为偏移地址，用地址 A + 地址 B 。

写作 `A-B` 。但是，如果外部也有一个叫做 B 的对象呢？这个 B 的值就存在二义性了。

我希望的是，`A-B` 这种写法可以对应于 C 语言中的 `A.B` 。C 语言中，要使用外部的地址偏移，需要写成 `A+B` 的写法。

可以使用斜杠“/”来访问对象中的对象。

```
A =
B = 1
= A
```

如果有一个对象叫 A ，它是这样的：

```
A =
存 1 到 #1
= A
```

那么，可以用以下形式访问对象 A 的第一个存储单元：

```
A + #1
```
