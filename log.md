以上语句，会将 1、2 压栈，遇到“相加”后，从栈中弹出两个栈顶元素，做加法后压栈。

在“相加”对象内部，原本的操作是这样的：

```
相加 = 
    #1 #2
    ADD #1 #2  ;汇编伪代码
= 相加
```

按照我原来的逻辑，1、2 应该对应于“相加”对象的栈内容。

相当于，在数组中的 i 号和 i+1 号位置放了参数，然后把 i（栈底指针）传给了对象“相加”，让 i 作为对象的栈底了。

```
[...][7:1][8:2]
       ^
     相加
```

然后将 #1 #2 相加，结果存 #1。这与将 #1 #2 出栈后相加，结果入栈的结果是相同的。

但是，此时栈指针尚未归位。需要手动让栈指针指向结果所在位置。之前要传递栈底指针 i 给对象，就需要额外一个单元保存 i 。

我认为可以使用指针行为模拟参数的出栈。

解释器在执行过程中，如果遇到“令”语句，会将“令”之后的所有词元依次解析。规则如下：

- 当遇到立即量时，将其压入数据栈。

- 当遇到对象时，将栈底和栈顶保存。在执行完成后，栈顶应该指向最后一个返回结果。

例如

```
令 *字符串* 格式的 *你好世界！* 打印 *整数* 格式的 12345 打印
```

解析过程如下：

1. “字符串”三个字依次入栈，保存第一个和最后一个入栈的元素的指针，命名为 I 和 J

2. 遇到对象“格式的”，将其栈底指针设置为 I ，栈顶指针设置为 J

3. 执行对象“格式的”，I 到 J 之间的内容即为其参数。

4. 执行结束后，将结果依次压入以 I 为栈顶的栈。结果为“S”和 0。更新 J 为指向最后一个入栈的元素 0 的指针。
也就是将 J 作为结果栈的栈顶。

5. 从 J 继续读取程序，“你好世界！”五个字符依次入栈

此时栈的结构应该为：

“S” 0 “你” “好” “世” “界” “！”

6. 遇到对象“打印”，I 不变，更新 J 为当前最后一个入栈的元素的指针。

7. 执行对象“打印”，I 到 J 之间的内容即为其参数。

以字符串的形式打印“你好世界！”

8. 执行结束后，没有结果，令 J 指向栈底 I （结果栈的栈顶）

9. 从 J 继续读取程序……

自底向上的语法分析——LR 分析法

核心思想：移进-归约

> 每当移进一个符号，就查看栈中内容是否与某一个活前缀相同，若相同，则按照活前缀对应的规则进行归约，否则继续移进下一个符号。

我的语句的活前缀是怎样的？

LR 分析器需要一个分析栈，我原本的程序只依靠一个前缀匹配对应语句，把已经读取的信息给丢掉了。

要利用已经读取的符号信息，读取到一定长度的内容后，就可以唯一确定语句的类型了，接下来只需要等待语句的句柄出现（在我这里，相当于读取到了完整的语句），进行归约。
